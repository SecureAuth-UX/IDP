Getting Started
Welcome to SecureAuth Identity Store Administrative Portal!

Use SecureAuth Identity Stores to add and manage user identities,
such as contractors, temporary or seasonal workers, customers, and partners

SecureAuth Identity Store can be used to reduce costs and increase security for identities that you may have currently within enterprise data store like Active Directory or Azure AD.

The SecureAuth Identity Store (IDS) which is built upon the SCIM v2 standard, supports users, groups, password policies, password deny lists, and a rich robust privilege model. All capabilities are exposed as RESTful APIs for integration into your existing environments.

By using groups and privileges, you can control who has access to the data in SecureAuth Identity Stores. Organize permanent employees, such as leads, supervisors, managers, and Human Resources personnel into groups with specific privileges to manage user identities.

With the right privileges, you can upload a large number of user identities using CSV file uploads or manually add users. Once the user identities become active, users can log in to manage their personal profile information and safely access necessary applications.
